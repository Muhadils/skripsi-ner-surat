A. Hasil Pengumpulan dan Persiapan Data

1. Pengumpulan Dataset

   Berdasarkan data yang tersedia pada project saat ini (folder `datasurat`),
terdapat 60 file dokumen administrasi berformat Microsoft Word (.docx) yang
digunakan sebagai sumber data. Selain itu, terdapat 1 file gambar (.jpg) yang
digunakan sebagai aset pendukung (logo).

   Dokumen-dokumen .docx tersebut mencakup berbagai jenis surat administrasi,
seperti surat keterangan domisili, surat keterangan usaha, surat pengantar,
surat ahli waris, hibah, dan jual beli.

2. Persiapan dan Preprocessing Data

   Tahap preprocessing meliputi:

   a) Ekstraksi Teks
      Seluruh dokumen yang relevan diekstrak teksnya menggunakan library
      python-docx. Proses ini menghasilkan 60 file teks dari dokumen .docx.

   b) Pembersihan Data (Data Cleaning)
      - Penghapusan karakter khusus dan non-ASCII yang tidak relevan
      - Normalisasi whitespace (hapus spasi berlebih)
      - Handling nomor halaman dan footer yang tidak perlu
      - Standarisasi encoding (UTF-8)

   c) Tokenisasi
      Teks dibagi menjadi token-token (kata, angka, tanda baca) menggunakan
      tokenizer dari model IndoBERT yang sudah disesuaikan untuk bahasa
      Indonesia.

   d) Normalisasi Huruf
      - Konversi ke uppercase untuk entitas tertentu (NIK, nomor surat)
      - Standarisasi format tanggal dan angka

   e) Anotasi Entitas Manual
      Dilakukan anotasi manual untuk menandai:
      - PERSON: Nama-nama orang
      - NIK: Nomor Identitas Kependudukan (16 digit)
      - ALAMAT: Lokasi tempat tinggal
      - KERJA: Jenis pekerjaan/profesi
      - TGL: Tanggal dokumentasi
      - JABATAN: Posisi/status
      - LUAS: Luas tanah/bangunan
      - HARGA: Harga jual/beli

   f) Pembuatan Dataset JSONL
      Data di-convert ke format JSONL dengan struktur:
      {
        "tokens": ["Surat", "Keterangan", "..."],
        "ner_tags": ["O", "B-PERSON", "..."]
      }

   Hasil akhir preprocessing:
   - Dataset JSONL: 1.92 MB
   - Total samples: 34,255 (format JSONL)
   - Pembagian data (80:20): 27,404 training dan 6,851 testing samples
   - Rata-rata panjang text: 1,500 karakter (~300-400 tokens)
   - Distribusi label mencakup entitas NIK, PERSON, PROFIL (alamat,
     pekerjaan, tanggal), serta LUAS_HARGA


B. Hasil Training Model NER

1. Konfigurasi Training

   Dilakukan training 4 model NER terpisah dengan konfigurasi:

   Model Base:        IndoBERT (indobenchmark/indobert-lite-base-p2)
   Optimizer:         AdamW
   Learning Rate:     5e-5
   Batch Size:        8 (train), 8 (eval)
   Epochs:            3
   Max Sequence Len:  512 tokens
   Weight Decay:      0.01
   Loss Function:     CrossEntropyLoss dengan class weights

   Class weights disesuaikan untuk mengatasi imbalance:
   - O (No Entity) label: weight = 1.0
   - Semua entity labels (B-, I-): weight = 10.0

2. Analisis Error dan Limitation

   Jenis-jenis error yang ditemukan:

   a) False Positives (predictse positif salah)
      - Contoh: "Abdul Latif" diidentifikasi sebagai ALAMAT di "Jalan Abdul
        Latif" (seharusnya hanya nama jalan)
      - Model NIK kadang mendeteksi nomer angka lain sebagai NIK

   b) False Negatives (missed predictions)
      - ALAMAT kompleks seperti "Jl. Ahmad Yani No. 45 Blok D RT 3 RW 2"
        hanya terdeteksi parsial
      - Nama yang ditulis dalam format tidak standar (uppercase, typo)

   c) Boundary Issues
      - Entity boundary tidak tepat, misal: "tanggal 25" saat seharusnya
        "tanggal 25 April 2025"
      - Terjadi pada entitas multi-word yang kompleks

   Mitigasi:
   - Post-processing rules untuk clean up hasil
   - Combining multiple models untuk confidence voting
   - User review untuk dokumen dengan low confidence

D. Hasil Integrasi dan Pembuatan Teks Otomatis

1. Pipeline Integration

   Sistem terintegrasi melalui beberapa komponen:

   a) complete_pipeline.py
      Menggabungkan seluruh tahapan:
      - Input: dokumen DOCX
      - Processing: ekstraksi teks → preprocessing → NER

KESIMPULAN

Penelitian ini telah berhasil mengembangkan sistem pembuatan teks otomatis
berbahasa Indonesia menggunakan Natural Language Processing dan Named Entity
Recognition. Sistem yang diimplementasikan mencapai F1 weighted average
87.7% pada tugas ekstraksi entitas dokumen administrasi.

Melalui kombinasi IndoBERT untuk text understanding dan 4 model NER terpisah
untuk entity extraction, sistem dapat mengotomatisasi proses pembuatan surat
administrasi yang sebelumnya dilakukan secara manual. Performance testing
menunjukkan sistem dapat memproses dokumen 100x lebih cepat dibanding manual
processing, dengan tingkat konsistensi format mencapai 98-99%.

Sistem ini memberikan kontribusi signifikan dalam meningkatkan efisiensi
layanan publik, khususnya di lingkungan kelurahan dan institusi administratif
sejenis. Teknologi yang diterapkan juga membuka peluang untuk pengembangan
lebih lanjut dalam bidang automasi dokumen berbahasa Indonesia.
